# Documentation Quality Rubric

## Good documentation

- states what the product does
- states who should use it
- states what not to use it for
- includes data sources
- includes KPI formulas and grain
- includes data dictionary
- includes freshness
- includes trust and limitations
- includes run instructions
- links to evidence
- preserves caveats

## Bad documentation

- hides caveats
- says “data is clean” without DQ evidence
- lists KPIs without formulas
- lists tables without grain
- lacks run instructions
- lacks trust statement
- treats release-candidate as production

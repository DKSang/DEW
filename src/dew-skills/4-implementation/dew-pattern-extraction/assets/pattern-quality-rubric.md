# Pattern Quality Rubric

## Approved pattern

Use when:
- pattern was used successfully
- evidence exists
- context is clear
- trade-offs are documented
- reuse checklist exists

## Candidate pattern

Use when:
- pattern seems useful
- but evidence is limited
- project-specific assumptions remain

## Rejected pattern

Use when:
- pattern is too project-specific
- no clear reuse context
- caused more complexity than value

## Bad pattern

A bad pattern:
- has no context
- has no trade-off
- says “always do X”
- lacks evidence
- lacks when-not-to-use section

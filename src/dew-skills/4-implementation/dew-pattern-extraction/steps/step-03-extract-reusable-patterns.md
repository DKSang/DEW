# Step 03 — Extract Reusable Patterns

## Mandatory Rules

- Pattern must include context and trade-offs.
- Pattern must cite project evidence/artifacts.
- Do not create universal rule without when-not-to-use.

## Candidate pattern areas

- validation-before-architecture
- KPI feasibility before modeling
- source access/sample/schema before P1
- requirement gate before architecture
- medallion storage with minimal metadata
- ingestion schema snapshot + drift detection
- business rule catalog
- DQ evidence gate
- metric serving contract
- trust limitations statement
- story evidence manifest

## For each pattern

Record:
- problem
- context
- solution
- when to use
- when not to use
- trade-offs
- evidence
- reuse checklist

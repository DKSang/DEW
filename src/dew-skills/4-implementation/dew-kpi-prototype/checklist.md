# KPI Prototype Checklist

## Inputs

- [ ] KPI feasibility report loaded.
- [ ] Sample computation plan loaded.
- [ ] KPI selected.
- [ ] Source sample located.
- [ ] Schema snapshot or field list located.

## Prototype

- [ ] Prototype file created.
- [ ] Formula implemented as specified.
- [ ] Required fields checked.
- [ ] Grain checked.
- [ ] Output generated.
- [ ] Output saved.
- [ ] Caveats documented.

## Evidence

- [ ] Prototype path recorded.
- [ ] Sample output path recorded.
- [ ] Validation notes created.
- [ ] Evidence index updated.
- [ ] Decision log updated if a decision was made.
- [ ] Learning log updated if learning mode is enabled.

## Failure conditions

Fail if:
- fields are invented
- formula is silently changed
- sample output is fabricated
- output grain does not match and is not documented
- missing evidence is ignored

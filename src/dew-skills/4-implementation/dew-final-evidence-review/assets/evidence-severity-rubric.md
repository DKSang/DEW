# Evidence Severity Rubric

## Critical

Missing evidence blocks release.

Examples:
- no E2E validation
- no primary KPI output evidence
- no DQ evidence for critical rules
- no serving evidence for user-facing product
- no trust/limitations statement

## High

Usually blocks release unless caveated.

Examples:
- missing schema snapshot
- missing freshness evidence
- missing operational run log

## Medium

Can release with caveat.

Examples:
- non-critical metric evidence weak
- documentation evidence incomplete

## Low

Does not block release.

Examples:
- minor naming/documentation evidence issue

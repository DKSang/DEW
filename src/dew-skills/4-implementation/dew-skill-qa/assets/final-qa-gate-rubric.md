# Final QA Gate Rubric

## QA passed

Use when:
- no critical/high findings
- module catalog valid
- required skill folders exist
- cross-links valid
- gates/HALTs consistent

## QA passed with caveats

Use when:
- only medium/low findings remain
- caveats documented
- package can still be tested

## Fix required before package

Use when:
- high findings exist
- package may work but release would be risky

## Block release

Use when:
- critical finding exists
- module cannot be installed
- required workflow chain is broken
- missing required skill files

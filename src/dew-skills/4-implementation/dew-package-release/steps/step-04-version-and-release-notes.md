# Step 04 — Version and Release Notes

## Mandatory Rules

- Version must be explicit.
- Release notes must mention caveats.
- Do not call it stable if QA caveats are significant.

## Version options

A. 0.x pre-release  
B. 1.0 initial stable release  
C. Patch  
D. Minor  
E. Major  

Recommend one based on QA status.

## Create release notes

Include:
- version
- release type
- included skills
- workflow chain
- known caveats
- installation notes
- next improvements

## Update

Update:
- packaging-report.md
- final-release-notes.md

Ask:

# Release Risk Rubric

## Critical

Blocks release.

Examples:
- misleading KPI
- missing trust limitations
- privacy/security unresolved
- no E2E validation
- critical DQ failure

## High

Usually blocks release unless explicitly accepted.

Examples:
- incomplete evidence for primary KPI
- freshness unverified
- source reliability weak

## Medium

Release with caveat possible.

Examples:
- secondary metric incomplete
- documentation partial
- monitoring minimal

## Low

Release allowed.

Examples:
- cosmetic issue
- minor README improvement

# Story Validation Checklist

- [ ] Story loaded.
- [ ] Story status is review.
- [ ] Code review loaded.
- [ ] Review decision is approve or approve-with-caveats.
- [ ] Dev notes loaded.
- [ ] Evidence manifest loaded.
- [ ] ACs validated.
- [ ] Evidence validated.
- [ ] DQ requirements validated if relevant.
- [ ] Operational requirements validated if relevant.
- [ ] Review findings resolved or caveated.
- [ ] Done Gate approved by user.
- [ ] Sprint status updated accordingly.

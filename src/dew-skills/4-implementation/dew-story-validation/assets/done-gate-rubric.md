# Done Gate Rubric

## Done

Use when:
- ACs pass
- evidence valid
- review approved
- no critical/high findings remain
- DQ/operational requirements satisfied
- story file updated
- sprint status can update to done

## Done with caveats

Use when:
- ACs pass
- evidence exists
- medium/low caveats remain
- user accepts caveats
- downstream impact documented

## Return to dev

Use when:
- implementation incomplete
- evidence invalid
- AC partially fails
- fix is feasible

## Block

Use when:
- critical blocker remains
- design conflict exists
- evidence cannot be produced
- user decision required

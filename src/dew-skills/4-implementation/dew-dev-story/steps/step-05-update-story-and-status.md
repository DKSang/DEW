# Step 05 — Update Story and Status

## Mandatory Rules

- Dev cannot mark story done.
- Dev can move story to review only if implementation and evidence are complete.
- If evidence missing, keep story in-progress or blocked.

## Task

Update story.

## Update story sections

- Tasks completed
- Evidence Requirements
- Validation Commands / Approach result
- Files Created / Modified
- Completion Notes
- Caveats
- Status

## Status rules

If implementation complete and evidence produced:
- story status → Review
- sprint-status → review

If blocked:
- story status → Blocked
- sprint-status → blocked
- blocker documented

If incomplete:
- story status → In Progress
- sprint-status → in-progress

`[C] Continue to finalize dev`

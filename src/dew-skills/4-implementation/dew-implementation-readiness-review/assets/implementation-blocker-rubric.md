# Implementation Blocker Rubric

## Critical blocker

Use when:
- required design artifact missing
- source/KPI/model dependency unresolved
- no evidence requirement
- no validation approach
- no acceptance criteria
- dependency story not done

Status:
- blocked

## Major blocker

Use when:
- acceptance criteria are vague
- DQ requirement missing for data-changing story
- operational requirement missing for pipeline story
- caveat not carried forward

Status:
- needs revision or blocked

## Minor blocker

Use when:
- wording unclear
- file list incomplete
- documentation could be improved

Status:
- ready with caveats or needs revision

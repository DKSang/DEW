# Step 06 — CI/CD, Environments, and Handoff

## Mandatory Rules

- Environment strategy must be explicit.
- Implementation stories must inherit operational evidence requirements.
- Do not over-design production CI/CD for learning mode unless user chooses it.

## Task

Define environment and implementation handoff.

## Environment strategy

Options:
A. Local/manual only  
B. Local + lightweight CI  
C. Dev/test/prod environments  
D. Cloud workspace deployment pipeline  
E. Hybrid local-first + cloud target  

Ask user if current operational mode requires a choice.

## CI/CD handoff

Define:
- what tests must run
- source contract tests
- DQ tests
- lint/format if relevant
- deployment steps
- evidence generated by CI
- manual approvals

## Implementation handoff

Stories must include:
- run logs
- DQ evidence
- freshness evidence
- schema drift evidence
- incident/backfill notes
- runbook updates

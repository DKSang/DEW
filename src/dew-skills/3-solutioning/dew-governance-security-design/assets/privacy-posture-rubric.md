# Privacy Posture Rubric

## A. No sensitive data identified

Use when:
- data is public or non-sensitive
- no personal data
- no restricted internal data

## B. Low-risk internal data

Use when:
- data is internal
- no direct PII
- access should still be controlled

## C. PII/sensitive data present with controls

Use when:
- personal or sensitive fields exist
- access controls, minimization, and retention must be explicit

## D. High-risk data requiring governance review

Use when:
- regulated data
- sensitive personal data
- safety/financial/legal risk
- external exposure

## E. Unknown; block until reviewed

Use when:
- data sensitivity is unclear
- source content has not been inspected

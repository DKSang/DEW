# Self-Service Rubric

## No self-service; curated outputs only

Use when:
- trust is limited
- consumers should not derive arbitrary metrics
- MVP scope is narrow

## Limited self-service with documented metrics

Use when:
- analysts can filter/slice approved metrics
- formulas are documented
- dimensions are controlled

## Full semantic/metric layer

Use when:
- multiple teams consume metrics
- metric consistency matters
- reusable definitions are required

## API-first contract

Use when:
- applications consume data
- versioning and schema contracts matter

# Epic Readiness Rubric

## Ready

Use when:
- epic has clear goal
- design inputs exist
- story candidates are scoped
- dependencies are known
- evidence requirements are defined

## Ready with caveats

Use when:
- design inputs exist but caveats remain
- caveats are carried into stories
- user approves continuation

## Blocked

Use when:
- required design artifact is missing
- KPI/source/model/DQ dependency is unresolved
- evidence requirement is unclear

## Not applicable

Use when:
- epic is outside MVP scope
- user explicitly deferred it

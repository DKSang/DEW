# Source Priority Rubric

## Validate first

Use when:
- source supports primary KPI
- source is likely essential for MVP
- source risk must be resolved before architecture

## Validate later

Use when:
- source supports secondary KPI
- source useful but not MVP-critical
- source can be deferred without blocking architecture

## Exploration only

Use when:
- source may be useful for learning
- source is not reliable enough yet
- source is not required for MVP

## Reject early

Use when:
- source does not support KPI/product needs
- source is clearly inaccessible
- license or privacy risk is unacceptable
- better source exists

## Blocked

Use when:
- source is likely important
- but access/owner/legal/technical constraints block validation

# Source Validation Status Rubric

## P1 — Use in MVP

Use only when:
- access is verified
- sample exists
- schema snapshot exists
- grain is confirmed or bridge/mapping is approved
- freshness is known or caveated
- field coverage supports primary KPI or product requirement
- major DQ risks are known
- user approves promotion to P1

## P2 — Use later

Use when:
- source is valuable
- not required for MVP
- or evidence is partial but promising

## Exploration only

Use when:
- source is useful for learning or research
- not reliable enough for product use
- field coverage or trust is weak

## Rejected

Use when:
- source does not support product/KPI needs
- source is inaccessible
- schema/quality/grain is unusable
- legal/privacy risk is unacceptable

## Blocked

Use when:
- source is important
- but access, owner, legal, or technical issue blocks validation

# Requirement Readiness Rubric

## Ready

Use when:
- required artifact exists
- evidence exists
- decision gate is approved
- caveats are minor or accepted
- no hard HALT remains

## Ready with caveats

Use when:
- evidence is partial but sufficient for next phase
- caveats are explicit
- user accepts caveats
- downstream impact is documented
- revisit trigger is defined

## Blocked

Use when:
- primary KPI is still hypothesis only
- P1 source is not validated
- trust expectation is missing
- grain is undefined
- freshness is unknown and material
- evidence is missing
- hard HALT remains unresolved

## Not applicable

Use when:
- item is outside MVP scope
- user explicitly deferred it
- decision is logged

# HALT Resolution Checklist

Use this checklist before resuming from a HALT.

## Required

- [ ] HALT ID is identified.
- [ ] HALT type is identified.
- [ ] Trigger is documented.
- [ ] Risk is explained.
- [ ] Missing evidence is listed.
- [ ] Options are provided.
- [ ] Recommendation is provided.
- [ ] User chooses a resolution.
- [ ] Decision log is updated.
- [ ] Learning log is updated.
- [ ] Evidence is produced or caveat is approved.
- [ ] Current artifact frontmatter is updated.
- [ ] Resume condition is met.

## Allowed resolution states

- resolved
- resolved-with-caveat
- deferred
- returned-to-previous-phase
- scope-reduced
- blocked

## Not allowed

- resume without user decision
- resume without decision log
- resume with missing evidence unless caveat is explicitly approved
- mark phase complete with unresolved hard halt

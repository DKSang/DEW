# Phase Done Evidence Checklist

A DEW phase is done only if all required conditions are met.

## Universal done criteria

- [ ] Main artifact exists.
- [ ] Required decision gates are approved.
- [ ] Required evidence exists.
- [ ] Evidence is indexed.
- [ ] Decision log is updated.
- [ ] Learning log is updated.
- [ ] Open assumptions are labeled.
- [ ] Caveats are documented.
- [ ] Next-phase readiness is stated.
- [ ] No unresolved hard HALT remains.

## Evidence by phase

### Project Framing

- [ ] Project scope approved.
- [ ] Business decision identified.
- [ ] Consumer identified.

### KPI Discovery

- [ ] KPI formula defined.
- [ ] KPI grain defined.
- [ ] Required fields listed.
- [ ] Source dependency listed.
- [ ] Feasibility status assigned.

### KPI Feasibility

- [ ] Required fields checked.
- [ ] Sample computation exists or caveat approved.
- [ ] Sample output exists or KPI deferred.
- [ ] Feasibility status updated.

### Source Validation

- [ ] Access verified.
- [ ] Sample extracted.
- [ ] Schema snapshot captured.
- [ ] Grain confirmed.
- [ ] Freshness observed.
- [ ] DQ/profiling notes exist.
- [ ] Source priority approved.

### Requirement Gate

- [ ] Trust expectation approved.
- [ ] Grain approved.
- [ ] Freshness approved.
- [ ] KPI/source caveats approved.
- [ ] Architecture readiness approved.

### Architecture

- [ ] Validated context referenced.
- [ ] Architecture decision recorded.
- [ ] Technology trade-offs documented.
- [ ] Revisit trigger documented.

### Implementation

- [ ] Story has evidence requirements.
- [ ] Tests pass.
- [ ] Validation evidence exists.
- [ ] File list updated.
- [ ] Story status updated.

### Retrospective

- [ ] Learning captured.
- [ ] Reusable pattern extracted if applicable.
- [ ] Open issues documented.

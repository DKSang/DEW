# Project Brief Validation Rubric

## Decision Readiness

- Does the brief identify the project type?
- Does it identify target users or data consumers?
- Does it identify the decision supported?
- Does it distinguish assumptions from facts?

## Data Engineering Readiness

- Does it list initial KPI hypotheses?
- Does it list initial source hypotheses?
- Does it avoid claiming KPI/source feasibility prematurely?
- Does it state what evidence is required before architecture?

## Learning Readiness

- Does it include learning objectives?
- Does it identify which FDE lifecycle concepts the project will practice?
- Does it create useful next steps?

## Failure Conditions

The brief fails validation if:
- it jumps directly to architecture/tooling
- it treats unverified sources as accepted
- it treats KPI hypotheses as verified
- it lacks target consumer or decision context
- it has no MVP boundary
